# Proceso de Entrevistas

## Introducción

La siguiente comprende una serie de pasos a seguir, objetivos y sugerencias para realizar el proceso de entrevista a posibles candidatos de **OnePhase**.

## Panorama General

El proceso de entrevista a un candidato ocurre de la siguiente manera:

- Prefiltro:
Este es el primer contacto con el candidato. En esta etapa, el candidato será sometido a una serie de preguntas que puede ir de lo general a lo específico a manera de reconocimiento. Esta es la etapa en la cual se decidirá si el candidato aporta el valor suficiente para ingresar a la siguiente etapa.

- Entrevista Técnica:
En base a lo recopilado en el prefiltro, el candidato ahora pasará a manos de un experto técnico, el cuál tendrá la tarea de realizar un análisis del perfil del candidato usando toda la información proporcionada por el mismo (hoja de vida, perfil de github, información recopilada en el prefiltro, etc...) y en base a dicho análisis, se deberá establecer una serie de preguntas que van de lo general a lo específico. La cantidad de preguntas puede rondar aproximadamente de 15 a 20 y no debe tomar más de 20 minutos. La entrevista será realizada por teléfono. 

  Ejemplos:
  - ¿Cuál fue el último proyecto que realizaste y con qué tecnologías lo hiciste?
  - ¿Qué es MVC y para qué funciona?
  - ¿Cuál es la diferencia entre API y API RESTful?
  - ¿Cuá es tu experiencia con Docker?

  Las preguntas deberán ser contestadas desde la experiencia del candidato.

- Git:
En caso de que la entrevista técnica no haya tenido el éxito deseado, el candidato tendrá una última oportunidad de demostrar sus habilidades con un ejercicio que nos servirá para medir las capacidades del mismo. El ejercicio comprende una serie de pasos dependiendo del stack técnico del candidato, sin embargo cada ejercicio deberá tener un nivel de dificultad estándar.

- Entrevista Presencial:
Este es el último y probablemente más importante paso del proceso. Nuevamente el experto técnico se tomará la tarea de realizar esta entrevista de manera presencial y, recopilando la información de la entrevista técnica, deberá formular de 3 a 5 preguntas que abarquen temas técnicos con un mayor grado de complejidad y que estén enfocados 100% en los skills y el puesto al que el candidato esté aplicando. _Tiempo estimado de duración: 30min._

  Adicionalmente, el candidato será presentado con un ejercicio de lógica de programación, el cuál tendrá que resolver utilizando una computadora proporcionada por el equipo de OnePhase. _Tiempo estimado de duración: 30min._

  Ejemplos:
  - Analizar un bloque de código y responder cuál sería la salida de dicho código si este fuese ejecutado.
  - Describir con código la manera para crear un test unitario y ejecutarlo.
  - Crear un componente que reciba propiedades y las mute.
  - Analizar un bloque de código e identificar los errores que este contiene.
